## US-003: Frame Extraction Module — COMPLETED
- Implemented frame extraction using ffmpeg-python's fps filter at configurable intervals (1 frame every N seconds)
- Frames saved to {output_dir}/frames/<timestamp_seconds>.jpg (e.g., 000003.jpg for 3s mark)
- Video duration auto-detected via ffmpeg.probe()
- Logs extraction progress with frame count, duration, and elapsed time
- Returns list of extracted frame Paths for downstream pipeline stages
- Typecheck (mypy), lint (ruff), and 19 existing tests all pass

## US-004: Local Vision Analysis — COMPLETED
- Implemented VLM-based frame analysis using mlx_vlm (load + generate) with configurable model path
- Accepts frames as list[Path] or directory path, resolves .jpg frames automatically
- Batch processing with configurable batch_size (default 4), mx.eval() and gc.collect() between batches for memory stability
- Generates timestamped JSON event log with "active", "description", "timestamp", and "frame" fields
- Graceful fallback for non-JSON VLM output; per-frame error handling without aborting the batch
- Saves vision_log.json to output directory for downstream pipeline stages
- 9 new unit tests (28 total passing); typecheck (mypy) and lint (ruff) clean

## US-005: Script and Cut-List Generation — COMPLETED
- Implemented full cut-list generation module in src/autovideo/generate.py
- Loads vision_log.json and filters out inactive/idle frames
- Groups active frames into continuous segments with configurable gap threshold
- Supports LLM-based generation (Qwen3 via mlx_lm) for enhanced voiceover and cut decisions
- Falls back to deterministic mode when LLM is unavailable (builds voiceover from frame descriptions)
- Outputs cut_list.json with "keep" timestamp ranges and "voiceover" script
- 17 new unit tests (45 total passing); typecheck (mypy) and lint (ruff) clean
- Aider review passed with suggested pattern added to AGENTS.md

## US-006: Automated Assembly — COMPLETED
- Implemented full assembly module in src/autovideo/assemble.py
- Loads cut_list.json, parses "keep" timestamp ranges, slices original video with MoviePy subclipped()
- Clamps ranges to video duration, skips invalid ranges (end <= start, negative start)
- Concatenates subclips with concatenate_videoclips(), renders output with libx264/aac codecs
- Cleans up temp directory (output/temp) on successful render
- 14 new unit tests (59 total passing); typecheck (mypy), lint (ruff), and aider review all pass

## US-007: Audio Extraction and Processing — COMPLETED
- Created src/autovideo/audio.py with extract_audio() using ffmpeg (aac codec, no video stream) and mix_voiceover() using ffmpeg amix filter with configurable voiceover volume
- Updated assemble.py run() with voiceover_path parameter; loads voiceover as AudioFileClip, scales volume to 70%, mixes with original audio via CompositeAudioClip
- Graceful degradation: voiceover failures logged as warnings; video rendered without voiceover rather than aborting
- Handles video clips with no audio track (voiceover used directly) and voiceover longer than video (clipped to duration)
- 10 new unit tests (69 total passing); typecheck (mypy), lint (ruff), and aider review all pass

## US-008: Output Validation and Verification — COMPLETED

- Uses ffmpeg.probe() to extract video/audio codec info, duration, resolution, and frame rate
- Reports file size in bytes and MB, logs all metadata fields for human audit
- Raises FileNotFoundError if output video is missing, ValueError if duration is zero or negative
- 15 new unit tests (84 total passing); typecheck (mypy), lint (ruff), and aider review all pass
